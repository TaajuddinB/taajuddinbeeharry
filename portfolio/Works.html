DROS Project


DROS online school platform using .net framework, c# language in Visual Studio 2019 platform

Case Study:
The Covid-19 pandemic situation prevailing across the globe and in our country has had a major impact on several sectors of our economy. Lockdowns and restrictions have resulted in the shutdown of many businesses. Both private and public sectors try to redouble their efforts to mitigate the economic impacts of the Covid-19 pandemic. Academic institutions have had to quickly adjust and migrate from traditional learning approach to an Online Teaching and Learning model.

Since the last confinement, we have noticed a sudden rush from both parents and students trying to get access to the right learning resources on social media. For this purpose Digital Repository Online System (DROS) a Mauritian institution wants to come up with an online platform that will act as a central repository for learning materials to assist students in their studies. One of the main objective of DROS is to gather learning resources from tutors across the island and share them with students. Likewise, there is an urgent need to develop an appropriate web application to cater for all the requirements as described below.

DROS comprises of an administrator and two other major type of users namely Tutors and Students. To have full access to the various functionalities of the system, both student and tutor must be authenticated users.

List of functionalities:
• F1 Tutor registers
• F2 Tutor adds course [Name, choose grade, description, aims etc…]
• F3 Tutor posts materials [syllabus, lectures, tutorials, assignments, links, exam… ]
• F4 Tutor views list of courses
• F5 Tutor amends course details
• F6 Tutor removes a course
• F7 Tutor views list of students
• F8 Tutor views/activates/deactivates subscription request (email notification)
• F9 Tutor/Student views reviews
• F10 Student searches for a course (display only basic details)
• F11 Student registers
• F12 Student amends profile
• F13 Student views list of all courses
• F14 Student: Advance Search for a course by grade / name, starting character
• F15 Student subscribes for a course (send request and email to tutor)
• F16 Student posts review
• F17 Student views/downloads course materials
• F18 Admin views list of tutors
• F19 Admin views list of students
• F20 Admin search for users
• F21 Admin freezes/unfreezes user account
• F22 Admin adds/amends course grade
• F23 Admin stats: Viewing number of students / tutors / courses
• F24 Student / Tutor / Admin log in
• F25 No access to course materials where students are not subscribed
• F26 Use of session to track users: custom message to greet user
• F27 Appropriate sign out measures
• F28 Displaying dynamic advertisements on the site
